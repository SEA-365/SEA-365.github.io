Java-String类常用方法源码分析

Java-String类常用方法源码分析

一、常用方法概述

  • 在刷LeetCode过程中，涉及字符串的操作时，发现自己经常这样想：jdk里边会不会有这个功能了，找找看吧！

  • 在开发过程中这样想当然没有大问题，因为一般情况下，jdk实现的操作肯定比我自己写的代码会简洁高效的；

  • 但是在刷题的过程中，总是想着调用jdk已经实现的方法，存在一个问题：当你写完一个算法题后，让你分析你写的代码，时间复杂度怎么样？是否开辟了额外的空间？

  • 因此，在刷题过程中，尽量避免在核心部分直接调用jdk已经实现的方法，除非已经了解了该方法的实现。

  • 常用方法：

    方法名称 描述
    equals(String) 比较两个字符串内容是否相同
    compareTo(String) 比较两个字符串的大小（ASCII码/Unicode码）
    indexOf(int) 字符串中第一次出现某个字符的位置（没有返回-1）
    indexOf(String) 字符串中第一次出现某字符串的位置（没有返回-1）
    replace(char, char) 将字符串中所有的字符A替换成字符B
    split(String) 以某字符串作为分隔符，分隔字符串，返回分割后的字符串数组
