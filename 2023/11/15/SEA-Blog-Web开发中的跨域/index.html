SEA-Blog-Web开发中的跨域

SEA-Blog-Web开发中的跨域

一、简介

  • 跨域（Cross-Origin）指的是在 Web 开发中，一个网页的资源请求（比如通过 JavaScript 发起的 HTTP 请求）来自于另一个域名（或者协议、端口）的情况。

  • 由于同源策略（Same-Origin Policy）的限制，跨域请求会受到浏览器的安全限制。

    • 同源策略是一种浏览器的安全策略，它要求网页中的脚本只能从相同的域（协议、主机和端口号相同）加载。
    • 这是为了防止恶意网站通过在其他网站上执行脚本来窃取数据，保护了用户的信息安全。

  • 跨域请求通常发生在以下情况下：

    1. 不同的域名： 当前页面所在的域与请求的域不同。
    2. 不同的子域： 子域名不同。
    3. 不同的协议： 请求的协议与当前页面的协议不同。
    4. 不同的端口： 请求的端口号与当前页面的端口号不同。

  • 在跨域的情况下，浏览器通常会拒绝执行某些跨域的操作，例如通过 Ajax 发起的跨域请求。

  • 为了允许跨域请求，可以采取一些方法，例如使用 JSONP、CORS（Cross-Origin Resource Sharing）、代理服务器等技术手段。
