二、加密方案

2.1 传统方案

  • 一般是使用哈希函数；
    • 需要一个好的哈希函数，例如消息摘要算法（MD5）、安全散列算法（SHA）等；
    • 为了避免哈希碰撞，在加密的过程中还需要进行“加盐”操作，这个“盐”可以是任意的字符(串)，盐值需要一个字段进行存储；

2.2 Spring Security提供的方案

  • Spring Security提供了多种密码加密方案，以增强应用程序的安全性，以下为一些常见的密码加密方案：
    • BCrypt：BCrypt 是一种基于哈希的密码加密算法，它是 Spring Security 推荐的默认密码编码器。BCrypt 使用工作因子来控制加密的强度，以便在需要时增加加密强度。
    • Argon2：Argon2 是一种密码哈希函数，被认为是非常安全的。Spring Security 5.0及更高版本引入了对Argon2的支持。
    • SCrypt：SCrypt 是另一种密码哈希函数，它在防止暴力破解攻击方面表现良好。Spring Security 5.0及更高版本也支持 SCrypt。
    • PBKDF2：PBKDF2（Password-Based Key Derivation Function 2）是一种基于密码的密钥派生函数，可以用于加密密码。它在 Spring Security 中也可用。
    • SHA-256 和 SHA-512：SHA-256 和 SHA-512 是常见的哈希算法，虽然不如BCrypt或Argon2安全，但在一些情况下仍然可以使用。请注意，不推荐明文存储密码。
    • Custom Password Encoder：如果您有自定义的密码加密需求，您可以实现自己的 PasswordEncoder 接口，以提供定制的密码加密算法。
