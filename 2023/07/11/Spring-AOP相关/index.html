Spring-AOP相关

Spring-AOP相关

一、概念

1.1 AOP简介

  • 全称：面向切面编程 Aspect Oriented Programming(AOP)
  • 简介：一种编程范式，直到开发者如何去组织程序结构；<==> 类比面向对象编程（OOP）
  • 作用：在不修改原代码的基础上为其增强功能。
  • Spring理念：无侵入式

1.2 核心概念

  • 根据下图：

    • 连接点（JoinPoint）：在程序执行过程中可以应用通知的特定点，粒度为执行方法、抛出异常、设置变量等；
      • 在SpringAOP中，理解为变量执行；
    • 切入点（Pointcut）：定义了在哪些方法或类上应用通知。
      • 通过使用切点表达式，可以根据方法的名称、参数、返回类型等来定义切点，如：
        • 一个具体的方法：com.sea.dao包下的BookDao接口中的无形参无返回值的save方法；
        • 匹配多个方法：所有的save方法、所有的get开头的方法、所有以Dao结尾的接口中的任意方法等等；
    • 通知（Advice）：在切入点处执行的操作，也就是共性功能，Spring AOP 中最终以方法的形式呈现。
      • 通知定义了在切入点何处执行什么操作；
      • Spring AOP支持一下几种类型的通知：
        • 前置通知（Before）：在方法调用之前执行。
        • 后置通知（After）：在方法调用之后执行（不考虑方法的结果）。
        • 返回通知（AfterReturning）：在方法调用之后执行，并获取方法的返回值。
        • 异常通知（AfterThrowing）：在方法抛出异常时执行。
        • 环绕通知（Around）：在方法调用之前和之后执行自定义操作。
    • 通知类：定义通知【通知呈现形式是方法，方法需要放在某个类中】的类。
    • 切面（Aspect）：描述通知与切入点的对应关系。

四、AOP切入点表达式

4.1 切入点和切入点表达式

  • 切入点：要增强的方法
  • 切入点表达式：要进行增强的方法【切入点】的描述方式

4.2 描述方式

  • 描述切入点既可以描述接口方法也可以描述该接口实现类中的方法，例如：

    • execution(void com.sea.dao.BookDao.update())
    • execution(void com.sea.dao.impl.BookDaoImpl.update())

  • 切入点表达式标准格式：

    • 动作关键字( 访问修饰符 返回值 包名.类/接口.方法名( 参数 ) 异常名 )
    • （1）动作关键字：描述切入点的行为动作，如execution表示执行到指定切入点；
    • （2）访问修饰符：public、private等，可以省略；
    • （3）返回值类型：指定要匹配的方法的返回类型，可以使用通配符 * 表示任意类型；
    • （4）包名.类/接口.方法名
    • （5）参数：指定方法的参数列表，可以使用通配符 * 表示任意参数；
    • （6）异常名：指定方法可能抛出的异常，可以使用通配符 * 表示任意异常，可以省略；

  • 使用通配符快速描述切入点：

    • *：单个独立的任意符号，可以独立出现，也可以左右前缀或者后缀的匹配符出现；

    • ..：多个连续的任意符号，可以独立出现，常用于简化包名与参数的书写；

    • +：专用于匹配子类类型；

    • 前置通知
    • 后置通知
    • 环绕通知【重点】
    • 返回后通知
    • 抛出异常后通知

5.1 前置通知

  • 在目标方法执行之前被触发的，用于执行一些准备性工作或者参数校验等操作。

5.2 后置通知

  • 在目标方法执行之后被触发的。

5.3 环绕通知

  • 在目标方法执行前后都能够进行处理，并且能够完全控制目标方法的执行。

  • 环绕通知可以决定是否调用目标方法，也可以在目标方法执行前后做一些额外的处理。

  • 环绕通知方法被调用时，Spring AOP框架自动将ProceedingJoinPoint类型的参数传递给通知方法。

    • 该参数提供了对目标方法的访问权限，包括方法调用、参数、返回值等信息。

  • 注意：环绕通知是最高优先级的通知类型！
