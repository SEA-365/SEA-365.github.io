数据库应用系统的设计与实现（MySQL+CentOS7环境）

数据库应用系统的设计与实现（MySQL+CentOS7环境）

一、需求分析

  • 收集和分析业务需求，以确定数据库的目标、范围和功能。这包括识别实体、属性和关系，并确定数据的完整性约束。

  • 需求如下：

  • 某银行准备开发一个银行业务管理系统，通过调查，得到以下的主要需求：

    • 这家银行在多个城市都有支行（即各个支行位于某个城市），每个支行有唯一的
      名字。银行要监控每个支行的资产。

      • 支行(名字， 所在城市， 资产)

    • 银行的客户通过其身份证号来标识。银行存储每个客户的姓名及其居住的街
      道和城市。客户都有帐户，并且可以存款和贷款。客户可能和某个银行员工发生
      联系，该员工是此客户的贷款负责人或银行帐户负责人。

      • 客户(身份证号， 银行账户负责人/贷款账户负责人-员工身份证号， 姓名， 居住的街道及城市)
      • 存款(存款号，账户号（外键），存款金额， 存款时间， 开户支行名字（外键）， 利率)

    • 银行员工也通过身份证号来标识。员工分为部门经理和普通员工（职位），每个部门
      经理都负责领导其所在部门的员工，并且每个员工只允许在一个部门内工作。每
      个支行的管理机构存储每个员工的姓名、 电话号码、 家庭地址及其经理的身份证号。

      银行还需知道每个员工开始工作的日期，由此日期可以推知员工的雇佣期。

      • 员工(身份证号，支行名字（外键）, 部门id（外键）， 所属经理的身份证号， 职位， 姓名， 电话号码， 家庭地址， 开始工作的日期)
      • 部门(部门id， 部门名称， 部门经理身份证号)

    • 银行提供两类帐户——储蓄帐户和支票帐户(账户类别)。一个客户可有多个帐户， 每个帐户被赋以唯一的帐户号。银行记录每个帐户的余额 、开户的支行以及每个
      帐户所有者访问该帐户的最近日期。另外，每个储蓄帐户有其利率，且每个支票帐户有其透支额。

      • 账户(账户号，客户身份证号（外键）， 账户类别， 余额， 开户支行名字（外键）， 最近访问日期， 利率， 透支额)

    • 每笔贷款由某个分支机构发放。每笔贷款用唯一的贷款号标识。银行需要知道每笔贷款所贷金额以及逐次支付的情况（银行将贷款分几次付
      给客户）。虽然贷款号不能唯一标识银行所有为贷款所付的款项，但可以唯一标识总贷款额。对每次的付款需要记录日期和金额；

      • 贷款(贷款号，支行名字（外键），账户号（外键），贷款申请时间， 贷款金额， 逐次支付的次数)
      • 付款记录(记录id， 贷款号（外键），付款金额， 付款日期)

三、数据库逻辑设计

  • 逻辑设计：把概念设计转换为关系模式，也就是数据库表的结构。这包括确定表格、列、主键、外键等等。

  • 如下：

    • 支行表（whm_BankBranch）

      • bankBranch_name：支行名字，主键；
      • city：支行所在城市；
      • asset：资产；

    • 客户表（whm_Custom）

      • custom_id：客户身份证号，主键；
      • employee_id：员工身份证号，外键；
      • custom_name：客户姓名；
      • custom_address：居住地址；

    • 员工表（whm_Employee）

      • employee_id：员工身份证号，主键；
      • bankBranch_name：支行名字，外键；
      • dept_id：部门id，外键；
      • job：职位；
      • employee_name：员工姓名；
      • call_number：电话号码；
      • employee_address：员工家庭住址；
      • JoinDate：开始工作日期；

    • 部门表（whm_Dept）

      • dept_id：部门id，主键；
      • dept_name：部门名称；
      • dept_leaderID：部门经理身份证号；

    • 账户表（whm_Account）

      • account_no：账户号，主键；
      • custom_id：客户身份证号，外键；
      • account_type：账户类别；
      • account_balance：账户余额；
      • bankBranch_name：开户支行名字，外键；
      • last_access：最近访问日期；
      • rate：利率；
      • overdraft：透支额；

    • 存款表（whm_Deposit）

      • deposit_no：存款号，主键；
      • bankBranch_name：支行名字，外键；
      • account_no：账户号，外键；
      • deposit_money：存款金额；
      • deposit_date：存款时间；
      • rate：存款利率；

    • 贷款表（whm_Loan）

      • loan_no：贷款号，主键；
      • bankBranch_name：支行名字，外键；
      • account_no：账户号，外键；
      • loan_money：贷款金额；
      • loan_createTime：贷款申请时间；
      • pay_times：逐次支付次数；

    • 付款记录（whm_PayRecord）

      • payRecord_id：记录id，主键；
      • loan_no：贷款号，外键；
      • pay_money：付款金额；
      • pay_date：付款日期；

四、数据库物理设计

  • 物理设计：选择适当的数据类型、索引、分区等等；

4.1 表结构设计

  • 具体设计如下：

    • 支行表（whm_BankBranch）

      字段名 描述 类型 主/外键
      bankBranch_name 支行名字 varchar(100) 主键
      city 支行所在城市 varchar(100) /
      asset 资产 bigint /

    • 客户表（whm_Custom）

      字段名 描述 类型 主/外键
      custom_id 客户身份证号 char(30) 主键
      employee_id 员工身份证号 char(30) 外键，参考员工表（whm_Employee）中的employee_id字段
      custom_name 客户姓名 char(50) /
      custom_address 居住地址 varchar(100) /

    • 员工表（whm_Employee）

      字段名 描述 类型 主/外键
      employee_id 员工身份证号 char(30) 主键
      bankBranch_name 支行名字 char(50) 外键，参考支行表（whm_BankBranch）中的bankBranch_name字段
      dept_id 部门id char(10) 外键，参考部门表（whm_Dept）中的dept_id字段
      job 职位 char(50) /
      employee_name 员工姓名 char(50) /
      call_number 电话号码 char(50) /
      employee_address 员工家庭住址 varchar(100) /
      joinDate 开始工作日期 timestamp /

    • 部门表（whm_Dept）

      字段名 描述 类型 主/外键
      dept_id 部门id char(10) 主键
      dept_name 部门名称 char(50) /
      dept_leaderID 部门经理身份证号 char(30) /

    • 账户表（whm_Account）

      字段名 描述 类型 主/外键
      account_no 账户号 char(50) 主键
      custom_id 客户身份证号 char(30) 外键，参考客户表（whm_Custom）中的custom_id字段
      account_type 账户类别 char(20) /
      account_balance 账户余额 bigint /
      bankBranch_name 开户支行名字 varchar(100) 外键，参考支行表（whm_BankBranch）中的bankBranch_name字段
      last_access 最近访问日期 timestamp /
      rate 利率，范围：0.00% ~ 100.00% decimal(5,2) /
      overdraft 透支额 bigint /

    • 存款表（whm_Deposit）

      字段名 描述 类型 主/外键
      deposit_no 存款号 char(50) 主键
      bankBranch_name 支行名字 varchar(100) 外键，参考支行表（whm_BankBranch）中的bankBranch_name字段
      account_no 账户号 char(50) 外键，参考账户表（whm_Account）中的account_no字段
      deposit_money 存款金额 bigint /
      deposit_date 存款时间 timestamp /
      rate 存款利率,范围：0.00% ~ 100.00% decimal(5,2) /

    • 贷款表（whm_Loan）

      字段名 描述 类型 主/外键
      loan_no 贷款号 char(50) 主键
      bankBranch_name 支行名字 varchar(100) 外键，参考支行表（whm_BankBranch）中的bankBranch_name字段
      account_no 账户号 char(50) 外键，参考账户表（whm_Account）中的account_no字段
      loan_money 贷款金额 bigint /
      loan_createTime 贷款申请时间 timestamp /
      pay_times 逐次支付次数 int /

    • 付款记录（whm_PayRecord）

      字段名 描述 类型 主/外键
      payRecord_id 记录id char(50) 主键
      loan_no 贷款号 char(50) 外键，参考贷款表（whm_Loan）中的loan_no字段
      pay_money 付款金额 bigint /
      pay_date 付款日期 timestamp /
