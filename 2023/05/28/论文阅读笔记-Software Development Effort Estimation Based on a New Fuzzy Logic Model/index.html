论文阅读笔记-Software Development Effort Estimation Based on a New Fuzzy Logic Model

论文阅读笔记-Software Development Effort Estimation Based on a New Fuzzy Logic Model

  • 中文标题：基于新的模糊逻辑模型的软件开发工作量估计
  • 来源：International Journal of Computer Theory and Engineering（国际计算机理论与工程期刊）

一、论文结构

  • 1.摘要（Abstruct）
  • 2.引言（Introduction）
  • 3.方法论（Methodology）
  • 4.结果（Result）
  • 5.结论（Conclusion）
  • 6.参考文献（Reference）

二、摘要

  • 研究背景：

  • 1.研究人员认为，传统的工作量估计技术需要进行长期的估计过程，而模糊逻辑可以提供一种替代方法来解决现有模型存在的问题。

  • 2.这篇论文描述了一个增强的模糊逻辑模型，用于估计软件开发工作量，并提出了一种应用模糊逻辑进行软件工作量估计的新方法。

  • 研究结论：

    • 研究结果表明，应用模糊逻辑的相对误差幅度的平均值（MMRE）明显低于其他模糊逻辑模型。这意味着使用增强的模糊逻辑模型进行软件工作量估计可以提供更准确的结果。

三、引言

  • 引言中分为三个部分：
    • 1.软件开发工作量估算（Software development effort estimation）
    • 2.模糊逻辑方法（Fuzzy logic approach）
    • 3.参数分析（Parameter`s analysis）

3.1 软件开发工作量估算（Software development effort estimation）

  • 软件度量：软件度量，尤其是软件估计，是基于对软件属性的测量，这些属性通常与软件开发的产品、过程和资源有关。

    • 这些测量的数据可以在项目管理的过程用作参考，以免成本超支和进度滞后等问题。
    • 软件工作量估计 是 软件度量 研究最广泛的领域之一.

  • 软件工作量估计：

    • 软件工作量估算模型主要分为两个大类：

      • (1)算法型：使用数学公式和统计方法来计算软件工作量。

        • 特点：这些模型通常基于历史数据和经验规则，通过输入软件项目的特征和属性，如代码行数、复杂性等，来计算预期的工作量。

        • 著名的算法型估算模型有：

          • Boehm的COCOMO（Constructive Cost Model）
          • Putnam的SLIM（Software Lifecycle Management）
          • Albrecht的功能点法

        • 适用场景：算法型估算模型提供了一种较为精确和可靠的估算方法，适用于有足够历史数据和准确输入参数的情况。

        • 局限性： 算法型模型提供了一种定量的估算方法，能够在一定程度上准确地估计软件工作量。然而，这些模型需要准确的输入参数，而在软件项目早期阶段往往难以获取准确的数据，因此存在一定的局限性。

        • 算法型估算模型的局限性 导致了对基于软计算的非算法技术的探索。

      • (2)非算法型：采用软计算（soft computing）方法，如模糊逻辑、神经网络和遗传算法等，来进行软件工作量估计。

        • 特点（优势）：通过处理模糊和不确定性的信息，能够更灵活地处理复杂的关系和处理非精确的输入数据。例如，人工神经网络在软件工作量估计中的应用，它能够学习先前的数据，建模复杂的关系，并对未见过的数据产生可接受的结果。
        • 适用场景：非算法型模型则更加灵活，能够处理不确定性和复杂关系，适用于数据不完全和环境不确定的情况。

3.2 模糊逻辑方法

  • 模糊逻辑是什么？

    • 一种处理不确定性的数学工具，也提供了一种处理非精确性和信息粒度的技术

  • 模糊逻辑模型使用了Lotfi Zadeh引入的模糊逻辑概念，这是模糊逻辑研究的重要里程碑。

  • 模糊推理由以下三个过程组成：

    • 模糊化过程 【模糊化】
      • 将客观术语转化为模糊概念的过程。
      • 将隶属函数应用于变量的实际值，以确定置信因子或隶属函数（MF）。
      • 模糊化使得输入和输出可以用语言术语来表达。
      • 例如，如果我们有一个表示温度的变量，我们可以使用隶属函数将实际温度值模糊化为”冷”、”温暖”或”炎热”等模糊术语。
    • 模糊规则推理 【推理出模糊规则，根据模糊规则得到模糊结果】
      • 根据规则条件的模糊化和置信因子传播，推导出规则的结论。
      • 多个规则可能被触发，推理引擎会为特定结果分配最大成员值的所有已触发规则。
    • 去模糊化过程 【将模糊结果进行去模糊化，得到具体的目标值】
      • 将模糊输出转化为具体的目标值。
      • 目的是从模糊的结果中提取出一个确定性的、可理解的值。
      • 常用的方法包括计算模糊输出的重心或使用模糊积分等数学运算来计算清晰值。

  • 其他学者在模糊逻辑方面的研究：

    • Gray 和 MacDonell：比较了功能点分析、回归技术、前馈神经网络和模糊逻辑在软件工作量估计中的应用。
      • 他们的结果表明，模糊逻辑模型取得了良好的性能，在准确性方面仅次于具有相当多输入变量的神经网络模型。
    • Fei 和 Liu：介绍了f-COCOMO模型，该模型将模糊逻辑（fuzzy logic）应用于软件工作量估计的COCOMO模式。
      • 由于在他们的研究中没有将f-COCOMO和其他努力估计模型之间的结果进行比较，因此前者的估计能力是未知的。
    • Roger：提出了一种模糊COCOMO模型，该模型采用模糊逻辑方法对软件努力驱动因素的不确定性进行建模；
      • 但没有提及所提出模型的有效性。

四、方法论

  • 使用了系统开发项目集。对于系统的41个模块，记录了每个模块的开发时间。同时，还记录了每个模块的耦合度（Dhama）、复杂度（McCabe）和代码行数等度量指标用于统计和分析。

五、结果

  • 作者通过：
  • 选择项目的子集，即10个项目；
  • 分别统计了 实际开发时间（Actual DT） 和 使用模糊逻辑估算的开发时间（Fuzzy Logic）；
  • 计算了每一个项目的相对误差幅度[Magnitude of Relative Error (MRE) ]
  • 最后计算出平均相对误差幅度（MMRE）；
  • 在这个过程中，使用了双侧高斯隶属函数进行模糊数据的处理，得出实验结果：应用该模糊逻辑的相对误差均值（MMRE）明显低于其他模糊逻辑模型。

六、结论

  • 本文提出了一种估算软件开发时间的新方法，关键点在于：模糊技术中的双侧高斯隶属函数用于软件开发时间的估计；
  • 作者通过收集数据对该方法的有效性进行了验证，得出模糊逻辑的优点和良好的推广性。
  • 主要优点：
    • 使用的模糊规则具有很好的可解释性；
    • 可以将专家知识（模糊规则）项目数据组合成一个通用框架，在软件评估中具有广泛的适用性。
